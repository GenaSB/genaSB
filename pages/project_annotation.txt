Аннотация к проекту автотеста для интернет-магазина "ИНТАНТ"
Автор: Геннадий
Дата: 12.05.2026
Цель проекта:
Автоматизировать end-to-end сценарий покупки товара с использованием
множественных фильтров и проверкой ключевых точек бизнес-логики.

Технологии:
- Python 3.12.0
- PyTest = 9.0.2
- Selenium = 4.41.0
- Page Object Model
- ООП

Тестируемый сценарий (Happy Path):
1. Открыть главную страницу
2. Нажать на кнопки всплывающих уведомлений «Ваш город», «Принять файлы cookie»
3. Нажать кнопку войти ввести логин пароль(авторизоваться)
4. Нажать кнопку каталог в выпавшей вкладке выбрать раздел товара «Ноутбуки»
5. Применить фильтры: Производитель "ASUS" → Объем SSD "512 ГБ" →
   Тип привода "Нет" → Тип графического контроллера "Интегрированный"→
Вес "От 1,5 до 2 кг"; Нажать кнопку "Показать"
6. Получить список товаров, выбрать первый
7. Добавить товар в корзину
8. Перейти в корзину, проверить соответствие товара и суммы
6. Нажать "Оформить заказ"
7. Заполнить поля (Контактный телефон, Имя, электронная почта) валидными данными
8. Нажать check boxes в разделе "Доставка и оплата"
9. Заполнить поле "Комментарий к заказу"
10. Нажать check box  "я принимаю условия публичной оферты и соглашаюсь с политикой обработки персональных данных. "
8. Нажать "Оформить заказ"

Особенности:
- Все локаторы вынесены в отдельные классы
- Реализован механизм ожиданий (WebDriverWait)
- Добавлены assert'ы на каждый ключевой шаг
- При падении теста сохраняется скриншот
